开始

Freebase API是一些列的HTTP API的集合，它提供了对存储在Freebase中的数据的读写方法。不同的API支持不同的场景，从而使得可以通过不同的方法对同样的Freebase数据进行访问。

API keys和权限验证

使用Freebase API 之前，你必须获得一个API key，也有一些用于调试的无需使用API key的配额。

一旦你已经有了一个key,你就可以通过使用URL的key参数，将它传给任何一个API：

https://www.googleapis.com/freebase/v1/search?query=bob&key=<YOUR_API_KEY>

如果想获得更多关于API keys的帮助，可以参考关于 Keys, security, and identity 的文档。
对于像MQL write（未本地化）这样需要额外的认证的API，请参见 请求的认证（未本地化）。

配给和使用限额

在使用限制中可以找到更详细的信息。
Freebase 允许一个用户在一天中，调用100k次读取数据的API（每24小时重新计算），调用10k次写数据的API.
如果你觉得对于你来说，100k次都不足以满足你的需求，那你可能需要考虑一下Freebase的数据集了，这样，你就能在本地访问Freebase了。
当然，你也可以在APIs Console(APIs 控制台)处，申请更高的额度。方法是到APIs Console，然后点击“Quotas”，再点击“Request more… ”，从而获得更多对Freebase API的调用额度。

鹏宝说：实际上操作起来要比这个复杂一点。首先这个网址经过了几次重定向，到达这里，然后你需要选择一个你的应用，点击进入，然后再左侧的“APIs&auth”中选择“APIs”，就能看到“Enabled APIs”中有“Freebase”这项，后面还有额度和已经使用了多少以及是否开启这个API的选项。鼠标悬停在那个标示用量的进度条上，在弹出的tip中，有“Request more”的链接，点进去是一个申请的页面，需要填写一些信息，比如你的应用的描述等等。
